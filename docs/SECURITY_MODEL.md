# Security Model

FlowFFmpeg is designed as a command compiler. Security depends on keeping workflow parsing, argument generation, process execution, and file validation as separate stages.

## Trust boundaries

### 1. Workflow input

Treat every workflow file as untrusted unless it was generated by a controlled system.

Validate:

- allowed node types;
- numeric ranges;
- input and output path policy;
- protocol policy;
- codec and format allowlists;
- maximum duration, dimensions, and frame rate.

### 2. Compiler output

The compiler should return an argument array, not a single shell command string.

Requirements:

- deterministic parameter order;
- no hidden shell expansion;
- no silent acceptance of unknown keys;
- explicit errors for invalid nodes;
- no execution side effects.

### 3. Execution layer

A separate runner should enforce:

- executable path pinning;
- working-directory restrictions;
- timeout and cancellation;
- CPU, memory, and disk limits;
- output overwrite policy;
- structured stdout and stderr capture;
- exit-code and artifact validation.

### 4. Generated media

A zero exit code does not prove the output is correct. The execution layer should verify:

- output file exists and is non-empty;
- duration is within expected bounds;
- expected video and audio streams exist;
- dimensions and frame rate match policy;
- temporary files are cleaned up safely.

## High-risk features

The following features require additional review before support:

- network URLs and streaming protocols;
- capture devices;
- arbitrary filter graphs;
- external subtitle or font paths;
- concat lists from user-controlled locations;
- shell execution wrappers;
- unrestricted hardware-device selection.

## Recommended path policy

1. Resolve paths to absolute canonical paths.
2. Reject paths outside configured input and output roots.
3. Use generated output names where possible.
4. Require explicit approval before overwriting.
5. Never treat a file extension as proof of media type.

## Logging

Every execution record should include:

- workflow checksum;
- compiler version;
- FFmpeg version;
- generated argument array;
- start and end time;
- exit code;
- output checksum;
- validation result.

## Non-goals

FlowFFmpeg does not claim to sandbox FFmpeg, inspect every codec vulnerability, or make arbitrary user-defined commands safe.
